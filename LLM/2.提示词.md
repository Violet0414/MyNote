#### 什么是提示词

​		**提示词是与LLM进行交互的核心**，它是用户输入给模型的指令、问题或者信息，**通过这些信息引导大模型生成特定的、符合期望的回复**。一个精心设计的提示词和和一个模糊的提示词得到的回复质量会有天壤之别。



- **模糊提示词**： “写点关于人工智能的东西。”
  - **结果**： 模型可能会生成一篇笼统的、介绍AI历史的科普文，也可能写一首关于AI的诗，结果不可控。
- **精确提示词**： “以科技博客作者的身份，写一篇800字左右的文章，介绍大语言模型在客服领域的应用，包括其三大优势和两个潜在挑战。目标读者是中小企业主。”
  - **结果**： 模型生成的回复会非常具体、有针对性，完全符合你的要求。

#### 提示词工程

1. **明确具体**

   - **不好**： “帮我写个营销方案。”
   - **好**： “为一款新的环保咖啡杯设计一个面向千禧一代的社交媒体营销方案，需要包括三个核心宣传点和两种主要的推广渠道。”

2. **提供参考范例**
   这是非常强大的技巧，称为 **“少样本提示”**。给模型一两个输入输出的例子，它能迅速理解你的模式和要求。

   ```
   请将中文口语转换成正式的书面语。
   
   例子1：
   输入： “这玩意儿忒好用了，你赶紧也整一个试试！”
   输出： “这款产品非常实用，建议您也尝试一下。”
   
   例子2：
   输入： “我跟你讲，那个地儿可远了，咱得早点动身。”
   输出： “那个地方距离较远，我们需要提前出发。”
   
   现在请转换这个：
   输入： “这饭馆的菜绝了，下次咱还来！”
   输出：
   ```

3. **赋予角色**
   让模型扮演一个特定角色，可以极大地改变其回复的风格和深度。

   - “你是一位经验丰富的历史教授，向大学生讲解罗马帝国衰亡的原因。”
   - “你是一个友好的客服机器人，需要耐心解决用户关于账户登录的问题。”

4. **分解复杂任务**
   对于复杂问题，将其拆解成几个简单的步骤，引导模型一步步思考。这通常与 **“思维链”** 技巧结合使用。

   ```
   问题： 如果小明有5个苹果，他给了小红2个，然后又买了3个，最后他有多少个苹果？
   请一步步推理：
   1. 开始时，小明有5个苹果。
   2. 给小红2个后，剩下 5 - 2 = 3个苹果。
   3. 又买了3个后，现在有 3 + 3 = 6个苹果。
   4. 所以，最后小明有6个苹果。
   ```

5. **使用分隔符**
   用三重引号`"""`、XML标签`<tag>`、节标题`###`等将指令、上下文和输入数据清晰分开，避免混淆。

------



#### 思维链

​	CoT是让模型**展示其推理过程**的提示技巧，而不仅仅是给出最终答案。这对于解决数学、逻辑推理等复杂问题至关重要。

- **零样本CoT**： 直接在提示词末尾加上 `“让我们一步步思考。”`
- **少样本CoT**： 在提供的范例中，就包含一步步推理的过程（如上面的苹果例子）。

---



### 最佳实践与注意事项

- **迭代优化**： 很少有一次就完美的提示词。根据模型的初次回复，不断调整和优化你的提示词。
- **保持礼貌**： 虽然模型不介意语气，但清晰的指令通常本身就包含礼貌和尊重的表达。
- **注意安全边界**： 不要试图让模型生成违法、有害或不道德的内容。好的LLM内置了安全护栏。
- **验证信息**： LLM可能会“幻觉”出看似合理但实际错误的信息（即“胡编乱造”），对于关键事实，务必进行核实。
