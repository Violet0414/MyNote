### RAGFlow本地部署

#### 安装WSL2

```powershell
# 以管理员身份打开 PowerShell
wsl --install
wsl --set-default-version 2
```

#### 安装 Docker Desktop

1. 下载 Docker Desktop for Windows
2. 安装时选择 **"Use WSL 2 based engine"**
3. 启动 Docker Desktop

#### 验证安装

```powershell
docker --version
wsl -l -v  # 应显示 "docker-desktop" 和 "docker-desktop-data"
```

---



#### 部署RAGFlow

**创建项目目录**

```bash
mkdir ragflow-windows
cd ragflow-windows
```

**下载 docker-compose.yml**
从 RAGFlow GitHub 仓库下载：

```bash
curl -O https://raw.githubusercontent.com/infiniflow/ragflow/main/docker-compose.yml
```

**修改配置文件**
编辑 `docker-compose.yml`，确保端口未被占用：

```yaml
# 修改端口映射（如果默认端口被占用）
ports:
  - "9380:9380"  # 前端端口
  - "3306:3306"  # MySQL
  - "6379:6379"  # Redis
```

**启动服务**

```bash
docker-compose up -d
```

---



在第一次安装docker-desktop时可能会出现无法启动的问题，界面提示需要执行**wsl --update**，但执行时会报以下问题：

**正在安装: 适用于 Linux 的 Windows 子系统无法启动服务，原因可能是已被禁用或与其相关联的设备没有启动。**



**问题可能的原因：**

1. 控制面板中有部分需要勾选的权限未被勾选。
2. 需要使用wsl --update --web-download开启强制下载。
