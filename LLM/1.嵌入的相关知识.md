#### 嵌入的技术原理

​		**工作流程：原始文本 → 分词 → 词汇ID → 嵌入层 → 向量表示 → 模型处理**

```python
import torch
import torch.nn as nn

class EmbeddingProcess:
    def __init__(self, vocab_size=50000, embedding_dim=768):
        self.embedding = nn.Embedding(vocab_size, embedding_dim)
    
    def text_to_vectors(self, text):
        # 1. 分词
        tokens = tokenizer.tokenize(text)  # ["Hello", "world"]
        
        # 2. 转换为词汇ID
        token_ids = tokenizer.convert_tokens_to_ids(tokens)  # [1234, 5678]
        
        # 3. 通过嵌入层获取向量
        embeddings = self.embedding(torch.tensor(token_ids))
        # 形状: [2, 768] - 2个token，每个768维向量
        
        return embeddings
```

#### 嵌入的维度特点

| 模型          | 嵌入维度 | 说明                     |
| :------------ | :------- | :----------------------- |
| **BERT-base** | 768维    | 每个token转换为768维向量 |
| **GPT-3**     | 12288维  | 更高的维度包含更多信息   |
| **Word2Vec**  | 300维    | 经典的词嵌入维度         |

**维度：即表示每个对象的向量长度，也就是向量的数量特征。**

```python
# 低维度嵌入 (2维)
cat_2d = [0.8, 0.2]    # 用2个数字描述"猫"
dog_2d = [0.7, 0.6]    # 用2个数字描述"狗"

# 高维度嵌入 (300维)  
cat_300d = [0.12, -0.45, 0.88, ..., 0.67]  # 用300个特征描述"猫"
dog_300d = [0.09, -0.38, 0.92, ..., 0.71]  # 用300个特征描述"狗"
```



| 维度大小      | 表示特点                       | 示例            |
| :------------ | :----------------------------- | :-------------- |
| **2-3维**     | 只能捕捉最基本关系，适合可视化 | `[x, y]` 坐标   |
| **50-100维**  | 能捕捉一些语义关系             | 简单词嵌入      |
| **300-500维** | 丰富的语义和语法关系           | Word2Vec, GloVe |
| **768维+**    | 复杂的上下文和细微差别         | BERT, GPT嵌入   |

​		俗话说过犹不及，**维度并不是越高越好**，其中还需要根据数据的规模进行选择，越高的维度相应需要的计算成本也会越高。
